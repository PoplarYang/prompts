# pp v0.1.5

## 中文

这版继续把 pp 往“轻量但真正好用的个人提示词工具”推进，重点补齐本地提示词、变量填写和原始文件跳转这几块日常高频能力。

### 新增

- 支持复制前填写变量：开启设置后，包含 `{{variable}}` 的提示词会先弹出填写面板；没有变量的提示词仍然直接复制。
- 支持来源过滤：可以在全部、GitHub、本地之间快速切换列表。
- 支持选择本地提示词文件夹：本地提示词不会和 GitHub 同步，适合放私有或离线提示词。
- 支持一键打开原始内容：GitHub 来源会打开对应 Markdown 页面，本地来源会打开本地 Markdown 文件。
- 新增本地提示词模板 Demo：用于快速生成和校验本地 Markdown 提示词格式。

### 改进

- 本地和 GitHub 提示词继续保留独立缓存，离线场景更稳。
- Release 流程会同时构建静态包、macOS 包和 Windows 安装包。

### 安装包

- macOS: `pp-desktop-macos-aarch64.dmg` 或 `pp-desktop-macos-aarch64.zip`
- Windows: `.exe` 安装包
- 静态版: `pp-v1.0.0-static.zip`

> macOS 包目前未签名、未 notarize，首次打开可能出现系统安全提示。

## English

This release keeps moving pp toward a lightweight but genuinely useful personal prompt launcher. The focus is local prompt libraries, variable filling, and quick access to the original prompt source.

### Added

- Optional variable filling before copy: when enabled, prompts containing `{{variable}}` placeholders open a small fill-in panel before copying. Prompts without variables still copy directly.
- Source filters for All, GitHub, and Local prompts.
- Local prompt folder picker: local prompts stay separate from GitHub sync and work well for private or offline prompt libraries.
- One-click open for the original source: GitHub prompts open the Markdown page, and local prompts open the local Markdown file.
- Local prompt template demo for generating and validating Markdown prompt files.

### Improved

- GitHub and local prompts continue to use separate caches for better offline behavior.
- The Release workflow builds the static package, macOS package, and Windows installer.

### Assets

- macOS: `pp-desktop-macos-aarch64.dmg` or `pp-desktop-macos-aarch64.zip`
- Windows: `.exe` installer
- Static build: `pp-v1.0.0-static.zip`

> The macOS package is currently unsigned and not notarized, so macOS may show a security warning on first launch.
