# pp v0.2 Acceptance Checklist

`pp` v0.2 adds public GitHub repository sync on top of the local v0.1 launcher.

## Default Repository

Default prompt repository:

```txt
https://github.com/PoplarYang/prompts
```

Default branch:

```txt
main
```

Default prompt directory:

```txt
prompts
```

## Demo Repository Content

The files under [demo-repo-content](../demo-repo-content) are ready to copy into the root of `PoplarYang/prompts`.

README.md

prompts/
  coding/

## Included

- Default repo is `https://github.com/PoplarYang/prompts`.
- Settings panel supports repo URL, branch, prompt directory, and sync-on-launch.
- Manual sync fetches Markdown prompt files from a public GitHub repository.
- Manual sync falls back to jsDelivr if GitHub tree API returns 403 or otherwise fails.
- GitHub sync reads `manifest.yaml` when present.
- GitHub sync parses Markdown files with YAML frontmatter.
- Synced prompts are cached in local browser storage.
- App loads cached synced prompts first for offline use.
- App falls back to bundled `prototype/prompts.json` when no cache exists.
- Failed sync does not break the current prompt library.

## Deferred

- Private repository auth.
- Native desktop shell.
- Global system shortcut.
- Native filesystem cache.
- Native app packaging.

## Commands

Build bundled fallback index:

```sh
python3 scripts/build_prompt_index.py
```

Verify v0.2 artifacts:

```sh
python3 scripts/verify_v0_2.py
```

Run the local app:

```sh
python3 -m http.server 8765
```

Open:

```txt
http://localhost:8765/prototype/
```
